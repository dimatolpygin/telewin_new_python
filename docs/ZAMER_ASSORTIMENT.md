# Замер этапа 34 — уточнения только по реальному ассортименту

**Повод**: UAT заказчика 2026-07-22. На «радиатор» бот отвечал «Для отопления, для машины или
охлаждения? И какой размер примерно?» — автомобильных радиаторов у хозяйственного магазина нет.
На «трубы» уходил в дымоход.

**Причина**: в tool-payload агента (`bot/ai/agent.py`) не было `группа`/`подгруппа`, а промт велел
уточнять ДО вызова поиска — варианты уточнения модель брала из общих знаний о мире.

**Что изменено** (только `bot/ai/agent.py`, тракт поиска не тронут):
1. `группа` и `подгруппа` в каждой позиции tool-payload.
2. `SYSTEM`: сперва `search_products` (`assortiment=true`), потом уточнение; уточнять только по
   видам из выдачи; внутреннюю разметку прайса покупателю не зачитывать.
3. **Предохранитель в коде** — ответ-вопрос без единого вызова поиска за ход форсирует
   `search_products` (`tool_choice=required`) и переспрашивает по реальной выдаче. Гард
   `_boltovnya` не форсирует поиск на «привет»/«спасибо»/командах.
4. Пустой `content` модели больше не подменяется заглушкой «Не понял запрос» — ещё одна итерация.

Золото: top-1 88% (35/40), hit@5 98%.

**Как это читать.** Абстейн / родовой / размерный точный / размер совпали с замороженным baseline
один в один. Разговорный 78% против 83% и золото 35/40 против 38/40 — это **известный разброс
приближённого вектор-канала** (HNSW approx + эмбеддинг запроса считается заново каждый прогон;
диапазон золота 35–38/40 зафиксирован ещё на этапе 22), а не эффект правки: `baseline_v2` меряет
гибрид напрямую и `bot/ai/agent.py` не импортирует вовсе. Изменён только `agent.py` — поиск, словарь,
гибрид, эмбеддинги не тронуты ни строкой (`git diff --stat`), поэтому регресс ранжирования исключён
по построению.
