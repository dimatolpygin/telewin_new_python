# Дорожная карта — Python-порт бота @NikaAi1Bot (ИИ-поиск по прайсу «Домашний мастер»)

7 этапов от пустого каркаса до протестированного бота с улучшенным поиском.
Каждый этап заканчивается **демонстрируемым результатом** (что можно потрогать руками).

Ключевая идея: **MVP-first**. Этап 1 = живой поиск из командной строки (уже ценность),
дальше наслаиваются БД, ИИ, Telegram, улучшения и тесты — без перепиливания ядра.

**Мотивация порта:** боевой TS-бот даёт 85% top-1; Python-эталон `poisk2.py` — 92%
(snowball-стеммер вместо `natural`). Плюс Python-стенд умеет два канала, которых нет в TS:
прямой поиск по **штрихкоду** (EAN 100% в новом прайсе) и recall-канал по **подгруппе Форы**
(категории 100%). Порт не переписывание — он структурно усиливает поиск.

**Хранилище:** Postgres (паритет с боевым ботом + задел под pgvector/семантику).
**Данные:** новый полный прайс `2026-07-18T11-29-30.xls` → `products.json` (11 864 позиции,
категории 100%, штрихкоды 100%).

> **Где смотреть прогресс**: текущий статус всех этапов — в [`STATUS.md`](STATUS.md).
> Внутри каждого этапа ниже — чекбоксы конкретных acceptance-пунктов. Закрытие этапа
> сопровождается git-тегом `stage-N-done`.

## Этап 0 — Каркас Python-проекта

**Цель**: пустой, но рабочий проект со всей инфраструктурой для следующих этапов.

## Этап 1 — Ядро поиска (паритет 92%)

**Цель**: детерминированный поиск из командной строки не хуже эталона `poisk2.py`.

**Что делаем**:
- порт `normalize.py` (snowball russian, гомоглифы, стоп-слова, `family_of`);
- порт `attributes.py` (`razobrat` уже есть в корне — переносим как есть);
- порт `search.py` — класс `Poisk` с каналами: прямой (артикул + **штрихкод**),
  словарь+атрибуты, **подгруппа Форы**; читает `products.json` (новый формат со штрихкодом);
- CLI `probe.py` — прогон золотого набора `zolotoy_nabor.json`.

**Критерии приёмки** (закрытие → tag `stage-1-done`):
- [x] `python -m bot.search.probe` печатает top-1/top-3 и список провалов
- [x] top-1 на золотом наборе ≥ 92% (37/40 = 92%, паритет с `poisk.py` подтверждён на общем файле)
- [x] запрос-штрихкод (13 цифр из прайса) возвращает ровно тот товар, канал = «штрихкод»
- [x] мусорный запрос (нет в прайсе) честно возвращает пусто (2/2)

## Этап 3 — ИИ-оркестрация

**Цель**: ИИ понимает живой запрос, вызывает поиск, называет только реальные числа.

**Что делаем**:
- `bot/ai/openrouter.py` — async httpx-клиент, ретраи (3, экспон. пауза), таймауты;
- `bot/ai/agent.py` — цикл tool-calling с `search_products`, системный промпт «Домашний мастер»;
- CLI `chat.py` для оффлайн-диалога (без Telegram).

**Критерии приёмки** (закрытие → tag `stage-3-done`):
- [x] на «нужны гвозди строительные 100» ИИ вызывает поиск и называет цену+наличие по двум точкам
- [x] ИИ не выдумывает числа: 0,17 руб / Микро 25450 / Берёз 31700 — точь-в-точь данные прайса
- [x] на неоднозначное «нужны саморезы» ИИ задаёт уточняющий вопрос, поиск не зовёт
- [x] сетевой сбой ретраится (3 попытки, 8.8с), невосстановимая 4xx — нет (1.2с, retry=False)

## Этап 4 — Telegram-слой

**Цель**: живой бот в Telegram поверх ядра поиска и ИИ.

**Что делаем**:
- `bot/main.py` — aiogram (polling), `/start`, `/reset`;
- per-chat очередь сообщений (от гонки быстрых сообщений);
- логирование каждого действия пользователя (username, id, текст, ответ бота);
- сессии диалога в Redis (TTL 30 мин, последние N сообщений).

**Критерии приёмки** (закрытие → tag `stage-4-done`):
- [x] `/reset` очищает историю в Redis; `/start` шлёт приветствие (подтверждено живым UAT)
- [x] реальный запрос в Telegram → бот отвечает ценой/наличием (подтверждено живым UAT)
- [x] два быстрых сообщения подряд обрабатываются по очереди (FIFO-lock, порядок [1,2] подтверждён)
- [x] в логах видно действие пользователя и ответ бота на русском (log_and_route + log_otvet_bota)
