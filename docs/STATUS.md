## Активная работа

🚧 **Этап 11 — Мерная линейка (Milestone 2).** Ветка `dev`. Только данные и замеры, код поиска
НЕ трогаем (безопасный первый шаг milestone). Задачи:
1. Собрать АБСТЕЙН-набор ≥40 чужих запросов (`ждём=null`) из находок (шампунь/масло для волос,
   зубная щётка, телевизор, ошейник-украшение и пр. ловушки хвоста) — источник `findings_svod.md` §1.
2. Собрать РАЗМЕРНЫЙ набор ≥30 кейсов с точным «ждём»-регэкспом (дюймы 1/2 vs 1½, М10/М12,
   саморез 4.8, цоколь E27/E14) — валидировать против прайса.
3. Снять baseline по 5 сегментам (top-1 / hit@5 / абстейн-точность) через `probe_batch`/`zamer_gibrid`
   → `docs/BASELINE_V2.md`. Золотой набор — контроль без изменений.
Верификация: прогон наборов + сверка каждого «ждём» против реального прайса.

Основа плана: UAT живым языком (товарный top-1 ≈77%, абстейн 60%) + прочёс прайса 5 субагентами
Haiku. Инвариант всех этапов milestone 2 — золотой набор ≥95% top-1 без регресса.

Новые инструменты (готовы к переиспользованию): `bot/search/probe_batch.py` (прогон произвольных
запросов через гибрид → JSON), `bot/search/dump_group.py` (выгрузка среза прайса по группе).
Отдельно: в `bot/ai/agent.py` добавлен предохранитель от ложного «нет в прайсе» без вызова
поиска (баг «масло пихтовое», коммит `af94e1e`) — не этап milestone, а точечный фикс.

Инфра гибрида (этап 10): отдельный pgvector-контейнер `telewin-test-pg` (:5434,
`pgvector/pgvector:pg16`), схема `telewin_test`, колонка `embedding halfvec(3072)` + HNSW.
Бот ходит на :5434 (`.env` PGPORT). Общий `postgres16` (:5432) НЕ трогали — там чужие проекты.
Пересчёт эмбеддингов: `python -m bot.embed_index --all` (нужен `OPENROUTER_API_KEY`).

## Протокол обновления

**При старте сессии**:
1. Прочитать этот файл.
2. `git tag -l "stage-*"` и сверить с колонкой «Закрывающий тег». Тег без записи в таблице = таблица устарела, обновить.
3. `git log --oneline -10` — что менялось последним.
4. Только после этого приступать к работе.

**При взятии этапа в работу**:
1. Обновить «Текущий этап» и «Активная работа».
2. Статус 🚧 в таблице.
3. Закоммитить отдельно: `chore(status): start stage N`.

**При закрытии этапа**:
1. Убедиться, что все чекбоксы в `ROADMAP.md` отмечены `[x]` и реально проверены.
2. Создать тег: `git tag stage-N-done -m "<описание>"`.
3. В таблице: ✅, «Закрывающий тег», «Последний коммит» (`git rev-parse --short HEAD`), «Дата закрытия».
4. Добавить запись в «Историю закрытий».
5. Указать новый «Текущий этап» и «Следующий шаг».
6. Закоммитить: `chore(status): close stage N`.

**Если этап заблокирован**:
1. Статус ⏸ в таблице.
2. В «Известные блокеры»: что блокирует, кого ждём, что нужно для разблокировки.
