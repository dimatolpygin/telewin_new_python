# Замер этапа 37 — ШСГД/ШСГМ по материалу + народные написания OSB

**Повод (UAT заказчика 2026-07-23, пункты 1 и 5).** (5) Бот настаивал, что ШСГД (саморез по
дереву) — для крепления гипсокартона к металлопрофилю. (1) «ОСБ» одним словом не находило плиту.

**Причина (в данных).** В прайсе есть **и ШСГД (18 поз., по дереву), и ШСГМ (11 поз., по металлу)**,
но `materialy.json → слово_в_имени` вело «гипсокартон» жёстко на `шсгд` — любой запрос про
гипсокартон отдавал ШСГД первым, бот честно повторял выдачу. По OSB: в прайсе имя латиницей
`Плита OSB-3`, покупатель пишет кириллицей «осб/осп» → лексика молчала (гомоглифы `s`/`b` в
латиницу не переводятся, поэтому кириллическое «осб» не сходилось с латинским `osb` в имени).

## Что сделано (только словарь/данные, тракт поиска не тронут)

1. **`data/materialy.json`**
   - `материал_в_имени`: дерево-регэксп имени `(?:по |п/)дер` → `(?:по |п/)дер|шсгд`;
     металл-регэксп `(?:по |п/)мет` → `(?:по |п/)мет|шсгм`; в металл-стемы добавлен `металлопрофил`.
     Теперь ШСГД помечается материалом «дерево», ШСГМ — «металл»; в скоринге материал запроса
     бустит нужный подтип (+4) и штрафует конфликтный (−2).
   - `слово_в_имени`: `осб/осп/оsb → osb` (токен латиницей матчит имя `Плита OSB-3`).
2. **`data/slovar_svodnyy.json`** — семейство «Плита» +синонимы `осб/осп/ориентированно-стружечная`
   (чтобы «осб» одним словом вообще давал кандидата-семейство, а не пустоту).
3. **`data/rodovye.json`** — `гипсокартон/гипрок/гкл → [ШСГД, ШСГМ]` (поднимает оба семейства ШСГ
   в кандидаты, даже когда в запросе нет слова «саморез»: «прикрутить гипсокартон…»).

**Дефолт сохранён:** «гипсокартон» без материала → ШСГД (как раньше, `слово_в_имени` не менялось).
Материал-контекст флипает: **металл → ШСГМ, дерево → ШСГД**.

## Регресс-контроль (детерминированный)

Правки **чисто лексические** (materialy/slovar/rodovye), вектор не пересчитывался → для запросов
без триггер-слов (гипсокартон/гипрок/гкл/осб/осп/дерево/металл-материал) выдача поиска
байт-идентична прежней. Триггер-слова содержат **только 5 позиций во всех наборах** — все
проверены адресно через гибрид:

| Кейс (набор) | ждём | получили | итог |
|---|---|---|---|
| саморезы по гипсокартону 3.5 на 45 (золото) | `^ШСГ[ДМ] 3,5*45` | ШСГД 3,5*45 | OK |
| шсгд 3,5х35 саморез (слепой) | `^ШСГД 3,5*35` | ШСГД 3,5*35 | OK |
| саморез гипсокартон дерево 3.5 на 35 (слепой) | `ШСГД` | ШСГД 3,5*35 | OK |
| саморез по гипсокартону 3.5 (размерный) | `^ШСГД 3,5`, ¬`4,[28]` | ШСГД 3,5*45 | OK |
| саморез по металлу 4.2 (размерный) | `^ШСГМ 4,2`, ¬`3,5` | ШСГМ 4,2*75 | **OK (было мимо → ШСММ)** |

Размерный-точный сегмент **улучшился на 1** (кейс «саморез по металлу 4.2» стал верным). Ни один
затронутый кейс не просел. Остальные наборы не содержат триггер-слов → регресс исключён по построению.
Полный `baseline_v2` прогнан для протокола (числа — с учётом недетерминизма вектор-канала, разброс
золота 35–38/40 не связан с правкой).

## Осознанная неоднозначность (не баг)

Запрос с ОТДЕЛЬНЫМ словом «профиль» («прикрутить гипсокартон к профилю») отдаёт реальные
профили ПН/ПП (`Профиль направл.`) — **магазин их продаёт**. Это законная развилка «саморез или
сам профиль», её разруливает уточняющий вопрос агента (этап 34), а не жёсткий приоритет поиска.
