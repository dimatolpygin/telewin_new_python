# Этап 6 — тестирование поиска субагентами

Честная оценка «видит ли ИИ каталог» на широком наборе **живых** формулировок, а не на
вылизанном золотом наборе. Запросы сгенерированы субагентами Haiku, размечены ожидаемым
товаром и провалидированы против прайса.

## Как собирали набор

- 6 субагентов Haiku, по кластерам из 11 групп прайса (`ref_*.txt` — реальные названия-якоря
  из `products.json`, по подгруппам). Каждый субагент писал ~18–28 реалистичных запросов
  (сленг, опечатки, родовые слова, размеры, вопросы про наличие) + 2–3 мусорных.
- Каждый товарный кейс размечен regex-меткой `ждём`, привязанной к реальному имени товара.
- **Валидация против прайса**: метка обязана совпасть хотя бы с одним именем из `products.json`,
  иначе кейс выбрасывается (скрипт слияния). Отброшено 7 из 162 (кривой regex/несуществующий хвост).
- Итог — `data/test_nabor.json`: **155 запросов = 142 товарных + 13 мусорных**, покрыты все 11 групп.

Прогон: `python -m bot.search.test_subagenty` (в проде поиск идёт с `use_podgr=True`, тут так же).

## Семейства-провалы и правки словаря

Разбор 35 провалов top-1 (из них 16 подхватываются в top-3, 19 промахнулись совсем).

### 1. Сантехника: путаница материала фитинга (главный провал, −6 кейсов)
Поиск не различает **полипропилен (ПП) / полиэтилен (ПЭ) / ПВХ / латунь** — родовое слово
«уголок/муфта/тройник/заглушка» уводит в чужой материал.
- «уголок 20мм полипропилен» → *Уголок ПВХ* (нужен *Уголок 20\*45", ПП*)
- «муфта для трубы ПП 20» → *Труба профильная* (нужен *Муфта 20, ПП*)
- «тройник пластик 20» → *Тройник … ц/ц/ц* мимо ПП-серии; «заглушка 110 канализация» → *Заглушка д/каб.канала*
- **Правка словаря**: завести семейства `муфта ПП / уголок ПП / тройник ПП` (синонимы:
  полипропилен, ППР, пластик) и `… ПЭ` (полиэтилен), с привязкой к подгруппам
  «Фитинги и полипропиленовая» / «…полиэтиленовая». Слово-материал должно бустить подгруппу.

### 2. Сленг дюймов и резьбы недокрыт (−3 кейса)
- «полудюйма» не сматчилось (в `sleng_razmerov.json` есть «пол дюйма»/«полдюйма», нет слитного «полудюйма»).
- «американка» → нужна *Кран шар. амер.*; «на 3/4», «гайка на 3/4» — размер как атрибут не всегда решает.
- **Правка**: добавить в словарь сленга форму `полудюйма → 1/2`; синоним `американка → амер.`.

### 3. Размерный канал слаб на кг/см/штуках (−5 кейсов, приём «размер» 70%)
Семейство находится, но выбирается не тот типоразмер:
- «цемент 25 килограмм» → *Цемент 2 кг*; «молоток 500 грамм» → *Молоток 200 г*;
  «рулетка 3 метра» → *Рулетка 2м*; «шланг для душа 175см» → поливочный шланг.
- Числа матчатся против атрибутов имени, но вес/длина в кг/см/метрах разбираются хуже, чем мм/резьба.
- **Правка (движок, не словарь)**: расширить `atributy_zaprosa`/`razobrat` на единицы кг, г, см, м
  (не только мм и `\d*\d`), и штрафовать несовпадение веса сильнее.

### 4. Инструмент: подтип и родовое (−8, но top-3 86%)
- «ключ на двадцать шесть» → шестигранник (число словами «двадцать шесть» не разобрано);
  «бита филипс» → *Bita Torx* (нет синонима филипс→PH/крест);
  «круг абразивный Р40 125» → лепестковый вместо отрезного (подтип круга).
- **Правка словаря**: `филипс/крестовая → PH`, `торкс → TX`; числительные словами
  («двадцать шесть»→26) — отдельная нормализация чисел.

### 5. Отсутствует порог «не уверен» → слабое отклонение мусора (3/13)
Поиск **всегда** возвращает лучшее семейство, даже на одном слабом слове. Из 13 мусорных
лишь 4 — честные переспамы («жидкий кран»→*Жидкое стекло*, «микроволновка»→*Печь-набор*,
«стеклянная дверь»→*СТОП-Дверь*, «кондиционер сплит»→*Система выравнивания плитки*: цепляется
за одно общее слово). Остальные 6 «мусорных» — методическая погрешность генерации: товар
фактически есть (радиатор, вентилятор, светодиодная лента, поролон, замок, петля), субагент
ошибочно счёл их отсутствующими.
- **Рекомендация (движок)**: ввести минимальный порог скора семейства для ответа; ниже порога —
  честное «не нашёл / уточните», а не лучший из слабых. Это поднимет и точность мусора,
  и доверие к выдаче. Канал подгруппы на top-1 не влияет (замер с `--no-podgr` идентичен),
  так что порог — правильнее, чем отключение recall-каналов.

## Приоритеты (по влиянию)

1. **Материалы фитингов ПП/ПЭ/ПВХ** — правка словаря, чинит худшую группу (Сантехника 50%).
2. **Порог уверенности + абстейн** — правка движка, чинит мусор и родовые переспамы.
3. **Единицы кг/г/см/м в разборе атрибутов** — правка движка, приём «размер» 70%→выше.
4. **Сленг: полудюйма, американка, филипс/крест, торкс** — точечные правки словаря.

Все правки — материал **следующей** итерации словаря/движка; этап 6 их только называет.

## Итерация правок №1 + №4 (материалы фитингов + сленг)

Сделаны две правки из плана, каждая под регрессионным замером на **обоих** наборах.

**Что сделано:**
- рулевой словарь `data/materialy.json`: слово-материал в запросе (`полипропилен/пп/ппр`,
  `полиэтилен/пэ/пнд`, `нержавейка/металлопласт`, `латунь`) бустит строки нужной подгруппы
  Форы; сленг-слово бустит токен в имени (`филипс/крестовая→PH`, `торкс→TX`, `американка→амер`);
- `search.iskat` — метод `rulevye()` + бусты в скоринге (материал +4, токен +3);
- `sleng_razmerov.json` — `полудюйма→1/2`.

**До/после:**

| Набор / метрика | До | После |
|---|---|---|
| Золотой top-1 / top-3 (регресс-гард) | 95% / 98% | **95% / 98%** (без просадки) |
| Слепой top-1 | 75% | **77%** (109/142) |
| Слепой top-3 | 87% | **89%** (126/142) |
| Сантехника top-1 / top-3 | 50% / 67% | **58% / 75%** |
| приём «сленг» top-1 | 43% | **57%** |
| приём «родовое» top-1 | 90% | **95%** |

Ни одна группа не просела; золотой набор держит 95% → правки безопасны.

**Что осталось недобитым в фитингах** (осознанно, без оверфита под тест):
материал теперь ведёт в верную подгруппу, но семейство внутри неё всё ещё путается на
конструкциях «муфта **для трубы** ПП» (головное слово «муфта», «труба» — зависимое →
поиск иногда отдаёт *Труба ПП* вместо *Муфта ПП*) и на формате размера («20мм» vs «20\*45»).
Это уже зоны №3 (единицы/формат размера) и разбор головного слова — следующий заход.
Константы буста (+4/+3) не подгонялись под тестовый набор намеренно: лексикон материал→подгруппа
обобщаем, а тюнинг весов под слепой набор был бы оверфитом на оценочные данные.
